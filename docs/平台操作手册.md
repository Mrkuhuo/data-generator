# 多数据源模拟数据生成器操作手册

## 1. 文档目的

本文档用于指导使用者通过平台完成以下工作：

- 创建模拟数据集
- 配置数据投递连接器
- 创建投递任务
- 手动执行或定时执行任务
- 查看执行结果与日志

如果你当前要联调 SQL Server 或 Oracle，请优先阅读 `docs/目标数据库联调指南.md`，里面包含本地容器环境、连接参数和联调脚本示例。

如果你是第一次使用，建议先按照“快速上手”章节走一遍，先跑通文件输出场景，再扩展到 HTTP、MySQL、PostgreSQL 或 Kafka。

## 2. 平台概念

平台由 4 个核心对象组成：

- 数据集：定义“生成什么样的数据”
- 连接器：定义“把数据投递到哪里”
- 任务：定义“哪个数据集投递到哪个连接器，以及如何运行”
- 执行记录：查看“每次运行的结果、摘要和日志”

推荐理解方式：

1. 先建数据集
2. 再建连接器
3. 再建任务
4. 最后执行并查看结果

## 3. 本地访问方式

如果你当前使用的是本地开发环境，可通过以下地址访问：

- 前端页面：`http://127.0.0.1:5173`
- 后端接口：`http://localhost:8888/api`

如果只是想快速体验平台，建议使用本地演示配置启动后端：

```powershell
cd backend
$env:SPRING_PROFILES_ACTIVE='local-demo'
mvn spring-boot:run
```

前端启动方式：

```powershell
cd frontend
npm install
npm run dev
```

## 4. 快速上手

这是最短路径，也是最推荐的首次体验方式。

### 4.1 第一步：创建快速示例数据集

进入“数据集”页面，点击右上角 `快速示例数据集`。

创建成功后，你会看到一个名为“示例用户行为数据”的数据集，里面已经内置了如下字段规则：

- `userId`：自增序列
- `city`：加权枚举
- `score`：随机小数
- `active`：布尔值
- `createdAt`：随机时间
- `profile`：对象嵌套字段
- `tags`：数组字段
- `email`：模板字段

### 4.2 第二步：创建快速示例连接器

进入“连接器”页面，点击右上角 `文件`。

系统会自动创建一个文件连接器，用于把生成数据写入本地文件。默认示例输出文件为：

`backend/output/example-preview.jsonl`

创建后建议点击连接器卡片上的 `测试` 按钮，确认页面显示“文件输出路径可写”。

### 4.3 第三步：创建快速示例任务

进入“任务”页面，点击右上角 `快速示例任务`。

系统会自动创建一个“示例投递任务”，并自动关联：

- 示例数据集
- 示例文件连接器

### 4.4 第四步：执行任务

在任务卡片上点击 `执行`。

执行成功后，系统会生成一条执行记录，并将模拟数据投递到文件中。

### 4.5 第五步：查看执行结果

进入“执行记录”页面，点击 `查看日志`。

你可以看到：

- 本次执行状态
- 生成条数
- 成功条数
- 失败条数
- 执行摘要
- 投递快照
- 详细日志

如果是文件输出，还可以直接打开输出文件查看生成结果。

## 5. 页面操作说明

## 5.1 总览

“总览”页面主要展示平台当前状态，包括：

- 连接器数量
- 数据集数量
- 任务数量
- 执行记录数量

适合快速确认当前平台是否已有配置和运行数据。

## 5.2 数据集页面

数据集页面用于定义生成规则，并支持实时预览。

### 5.2.1 主要字段说明

- 名称：数据集名称，例如“用户行为数据 v1”
- 分类：业务分类，例如“电商”“风控”“运营”
- 版本：数据集版本，例如 `v1`
- 状态：建议初期使用 `就绪`
- 说明：填写该数据集的业务用途
- Schema JSON：定义字段生成规则
- 示例配置 JSON：定义预览条数与随机种子

### 5.2.2 Schema JSON 示例

```json
{
  "type": "object",
  "fields": {
    "id": { "rule": "sequence", "start": 1, "step": 1 },
    "email": { "rule": "template", "template": "user-${id}@demo.local" },
    "score": { "rule": "random_decimal", "min": 80, "max": 99.9, "scale": 1 }
  }
}
```

### 5.2.3 示例配置 JSON 示例

```json
{
  "count": 5,
  "seed": 20260412
}
```

### 5.2.4 数据集支持的规则

当前已支持以下规则：

- `fixed`：固定值
- `sequence`：序列号
- `random_int`：随机整数
- `random_decimal`：随机小数
- `string`：随机字符串
- `enum`：普通枚举
- `weighted_enum`：加权枚举
- `boolean`：布尔值
- `datetime`：随机时间
- `object`：对象
- `array`：数组
- `reference`：引用其他字段
- `template`：模板拼接

### 5.2.5 数据集预览

保存数据集后，可以：

- 点击右侧 `生成预览`
- 或点击数据集卡片上的 `预览`

预览时可指定：

- 生成条数
- 随机种子

用途：

- 验证字段规则是否正确
- 确认生成结果是否符合业务预期

## 5.3 连接器页面

连接器页面用于定义数据输出目标。

当前支持的连接器类型：

- 文件
- HTTP
- MySQL
- PostgreSQL
- Kafka

### 5.3.1 通用字段说明

- 名称：连接器名称
- 类型：连接器类型
- 用途：平台固定按“目标端”处理，用于接收平台生成的数据
- 状态：建议选 `就绪`
- 说明：描述用途和目标环境
- 配置 JSON：不同类型的连接器配置内容不同

### 5.3.2 文件连接器

文件连接器适合首次体验，也是最稳定的方式。

示例配置：

```json
{
  "path": "./output/generated-users.jsonl",
  "format": "jsonl"
}
```

支持格式：

- `jsonl`
- `json`
- `csv`

使用建议：

- `jsonl`：适合追加和覆盖
- `csv`：适合追加和覆盖
- `json`：目前仅建议使用 `OVERWRITE`

### 5.3.3 HTTP 连接器

示例配置：

```json
{
  "url": "http://localhost:9000/mock/intake",
  "method": "POST",
  "batch": false,
  "timeoutMs": 5000,
  "headers": {
    "X-Source": "multisource-data-generator"
  }
}
```

字段说明：

- `url`：目标接口地址
- `method`：支持 `POST`、`PUT`、`PATCH`
- `batch`：
  - `true`：整批数据一次发送
  - `false`：逐条发送
- `timeoutMs`：超时时间，单位毫秒
- `headers`：自定义请求头

### 5.3.4 MySQL 连接器

示例配置：

```json
{
  "jdbcUrl": "jdbc:mysql://localhost:3306/demo_sink?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai",
  "username": "root",
  "password": "123456"
}
```

任务运行时还需要在 `target.table` 中指定目标表。

适用写入策略：

- `APPEND`
- `OVERWRITE`

### 5.3.5 PostgreSQL 连接器

示例配置：

```json
{
  "jdbcUrl": "jdbc:postgresql://localhost:5432/demo_sink",
  "username": "postgres",
  "password": "postgres"
}
```

任务运行时还需要在 `target.table` 中指定目标表。

适用写入策略：

- `APPEND`
- `OVERWRITE`

### 5.3.6 Kafka 连接器

示例配置：

```json
{
  "bootstrapServers": "localhost:9092",
  "acks": "all",
  "clientId": "mdg-local",
  "properties": {
    "security.protocol": "PLAINTEXT"
  }
}
```

任务运行时通常需要提供：

- `target.topic`
- `target.keyField`

适用写入策略：

- `APPEND`
- `STREAM`

### 5.3.7 连接器测试

创建或更新连接器后，建议点击：

- 表单区的 `测试当前连接器`
- 或卡片上的 `测试`

测试作用：

- 检查连接参数是否基本可用
- 输出最近测试状态
- 查看测试详情 JSON

## 5.4 任务页面

任务页面用于把“数据集”绑定到“连接器”，并定义运行参数。

### 5.4.1 主要字段说明

- 名称：任务名称
- 数据集：选择已创建的数据集
- 连接器：选择已创建的目标连接器
- 写入策略：定义投递行为
- 状态：建议使用 `就绪`
- 调度类型：手动、单次、定时
- Cron 表达式：仅在定时任务时需要
- 运行时配置 JSON：定义运行参数

### 5.4.2 运行时配置 JSON 示例

```json
{
  "count": 100,
  "seed": 20260412,
  "batchSize": 500,
  "target": {
    "table": "synthetic_user_activity",
    "topic": "synthetic.user.activity",
    "keyField": "userId"
  },
  "retry": {
    "maxAttempts": 2,
    "backoffMs": 1000
  },
  "schedule": {
    "triggerAt": "2026-04-12T14:00:00Z"
  },
  "note": "target.table 供 MySQL/PostgreSQL 使用，target.topic 与 target.keyField 供 Kafka 使用"
}
```

### 5.4.3 常用运行参数说明

- `count`：生成数据条数
- `seed`：随机种子，相同种子通常可复现相同结果
- `batchSize`：数据库批量写入大小
- `target.table`：MySQL/PostgreSQL 目标表
- `target.topic`：Kafka 目标 Topic
- `target.keyField`：Kafka 消息 Key 对应字段
- `schedule.triggerAt`：单次任务触发时间

### 5.4.4 调度类型说明

- `手动`：仅在你点击“执行”时运行
- `单次`：按 `schedule.triggerAt` 指定时间运行一次
- `定时`：按 Cron 表达式周期运行

### 5.4.5 Cron 表达式示例

每 5 分钟执行一次：

```text
0 */5 * * * ?
```

### 5.4.6 任务操作

任务创建后可执行以下操作：

- 编辑
- 执行
- 暂停
- 恢复
- 禁用
- 删除

如果是定时任务，还可以看到：

- 调度状态
- 下次触发时间
- 上次触发时间

## 5.5 执行记录页面

执行记录页面用于查看任务运行结果。

每条执行记录可以查看：

- 任务名称
- 触发方式
- 开始时间
- 结束时间
- 生成条数
- 成功条数
- 失败条数
- 执行摘要

点击 `查看日志` 或 `刷新日志` 后，还可以看到：

- 执行日志
- 投递快照
- 各阶段详细信息

常见日志含义：

- 接口触发执行已开始
- 数据集批次已生成
- 连接器投递完成

## 6. 推荐使用流程

## 6.1 推荐流程 A：先验证平台功能

适用场景：第一次接触平台。

步骤：

1. 创建快速示例数据集
2. 创建文件连接器
3. 创建快速示例任务
4. 执行任务
5. 查看执行记录和输出文件

## 6.2 推荐流程 B：数据库落库

适用场景：把模拟数据写入测试库。

步骤：

1. 创建数据集
2. 创建 MySQL 或 PostgreSQL 连接器
3. 在任务中设置 `target.table`
4. 写入策略选择 `APPEND` 或 `OVERWRITE`
5. 执行任务
6. 到数据库中验证结果

## 6.3 推荐流程 C：消息投递

适用场景：把模拟数据投递到 Kafka。

步骤：

1. 创建数据集
2. 创建 Kafka 连接器
3. 任务运行时配置 `target.topic`
4. 视情况配置 `target.keyField`
5. 写入策略选择 `APPEND` 或 `STREAM`
6. 执行任务并在执行记录中查看结果

## 7. 常见问题

## 7.1 页面显示 Network Error 是什么意思

含义：

- 前端发起了请求
- 但浏览器没有拿到后端返回

常见原因：

- 后端未启动
- 后端刚启动完成前请求已发出
- 本地开发代理未生效

现在前端已默认通过 `/api` 代理访问后端，刷新页面后通常可以恢复。

## 7.2 为什么“生成预览”按钮是灰的

因为当前还没有选中一个已保存的数据集。

处理方式：

1. 先创建并保存数据集
2. 在右侧数据集卡片中点击 `编辑` 或 `预览`
3. 再点击 `生成预览`

## 7.3 为什么任务页没有可选数据集或连接器

因为任务依赖前置对象。

处理方式：

1. 先创建数据集
2. 再创建连接器
3. 返回任务页刷新

## 7.4 为什么数据库或 Kafka 投递失败

常见原因：

- 连接地址不通
- 用户名密码错误
- 目标表不存在
- Topic 不存在或无权限
- 写入策略与连接器能力不匹配

建议：

- 先点击连接器 `测试`
- 再执行任务
- 到“执行记录”中查看详细日志

## 8. 操作建议

- 首次体验优先使用“文件连接器”
- Schema 先从简单对象开始，再逐步增加数组、模板、引用
- 运行时配置建议先只改 `count` 和 `seed`
- 复杂任务建议先手动执行，确认无误后再改成定时任务
- 出现报错时优先查看“执行记录”页面，而不是只看表单提示
