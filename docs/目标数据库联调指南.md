# SQL Server / Oracle 目标数据库联调指南

## 1. 文档目标

这份文档用于说明平台如何与 `SQLSERVER` 和 `ORACLE` 两类目标数据库做真实联调，并给出本地启动、连接配置、自动写入、结果校验与常见问题排查方式。

当前平台已支持：

- 后端连接类型：`MYSQL / POSTGRESQL / SQLSERVER / ORACLE`
- 前端可配置以上 4 类目标数据库
- 读取表结构并自动映射字段
- 将模拟数据写入目标表
- 返回写入前条数、写入后条数、净增条数、空值校验结果

## 2. 本地环境准备

### 2.1 SQL Server

仓库中的 `docker-compose.yml` 已提供 SQL Server 目标库：

```powershell
docker compose --profile sqlserver up -d sqlserver
```

默认连接信息：

- 主机：`127.0.0.1`
- 端口：`1433`
- 用户名：`sa`
- 密码：`MdgSqlServer123!`
- Schema：`dbo`

初始化示例表：

```powershell
docker exec -i mdg-sqlserver /opt/mssql-tools18/bin/sqlcmd -C -S localhost -U sa -P "MdgSqlServer123!" -i /work/init/01_demo_sink.sql
```

初始化后可得到：

- 数据库：`mdg_demo`
- 示例表：`dbo.synthetic_user_activity`

### 2.2 Oracle

仓库中的 `docker-compose.yml` 已提供 Oracle Free 目标库：

```powershell
docker login container-registry.oracle.com
docker compose --profile oracle up -d oracle
```

默认连接信息：

- 主机：`127.0.0.1`
- 端口：`1521`
- 服务名：`FREEPDB1`
- 用户名：`MDG_DEMO`
- 密码：`MdgDemo123!`
- Schema：`MDG_DEMO`

Oracle 相关说明：

- 镜像使用 `container-registry.oracle.com/database/free:latest`
- 首次启动通常明显慢于 MySQL / PostgreSQL
- 仓库会把 `infra/oracle/setup` 同时挂载到 `setup` 和 `startup` 目录
- 初始化 SQL 是幂等的，因此首次建库和后续重启都会自动兜底创建演示用户与示例表

初始化后可得到：

- Schema：`MDG_DEMO`
- 示例表：`MDG_DEMO.SYNTHETIC_USER_ACTIVITY`

## 3. 平台连接参数

### 3.1 SQL Server 示例

在平台的“目标数据源连接”页面填写：

- 数据库类型：`SQL Server`
- 主机：`127.0.0.1`
- 端口：`1433`
- 数据库名：`mdg_demo`
- Schema：`dbo`
- 用户名：`sa`
- 密码：`MdgSqlServer123!`
- JDBC 参数：`encrypt=true;trustServerCertificate=true`

### 3.2 Oracle 示例

在平台的“目标数据源连接”页面填写：

- 数据库类型：`Oracle`
- 主机：`127.0.0.1`
- 端口：`1521`
- 数据库名：`FREEPDB1`
- Schema：`MDG_DEMO`
- 用户名：`MDG_DEMO`
- 密码：`MdgDemo123!`
- JDBC 参数：留空

## 4. 自动化真实联调脚本

仓库提供脚本：

- `scripts/smoke-write-target.ps1`

脚本能力：

- 自动创建临时目标连接
- 自动创建写入任务
- 自动执行一次真实写入
- 自动读取目标表结构
- 返回写入前条数、写入后条数、净增条数、空值校验结果
- 默认自动清理测试连接与任务

### 4.1 SQL Server 示例

```powershell
.\scripts\smoke-write-target.ps1 `
  -DbType SQLSERVER `
  -TargetHost 127.0.0.1 `
  -Port 1433 `
  -DatabaseName mdg_demo `
  -SchemaName dbo `
  -Username sa `
  -Password 'MdgSqlServer123!' `
  -JdbcParams 'encrypt=true;trustServerCertificate=true'
```

### 4.2 Oracle 示例

```powershell
.\scripts\smoke-write-target.ps1 `
  -DbType ORACLE `
  -TargetHost 127.0.0.1 `
  -Port 1521 `
  -DatabaseName FREEPDB1 `
  -SchemaName MDG_DEMO `
  -Username MDG_DEMO `
  -Password 'MdgDemo123!'
```

### 4.3 返回字段说明

脚本会输出一段 JSON，重点字段如下：

- `connectionStatus`：连接测试是否成功
- `executionStatus`：本次写入是否成功
- `writtenRowCount`：本次写入条数
- `beforeWriteRowCount`：写入前条数
- `afterWriteRowCount`：写入后条数
- `rowDelta`：净增条数
- `nullValueCount`：非空字段校验发现的空值数量
- `blankStringCount`：非空字符串字段校验发现的空白字符串数量
- `importedColumnCount`：回读到的字段数量

## 5. 推荐联调顺序

建议按下面顺序执行：

1. 启动目标数据库容器。
2. 在平台中创建并测试目标连接。
3. 先用 `CREATE_IF_MISSING` 模式写入一张测试表。
4. 再从已存在表中选择刚创建的目标表，确认字段是否自动映射。
5. 执行写入，检查返回的写入前后条数与空值校验结果。
6. 最后再验证持续写入或定时写入能力。

## 6. 常见问题

### 6.1 SQL Server 连接时报证书错误

通常是 JDBC 加密握手导致，请确认 JDBC 参数包含：

```text
encrypt=true;trustServerCertificate=true
```

### 6.2 Oracle 启动后暂时连不上

Oracle 首次启动较慢，建议：

- 先等待容器日志出现 `DATABASE IS READY TO USE!`
- 再在平台点击“测试连接”

### 6.3 看不到刚创建的表

优先检查：

- `Schema` 是否填写正确
- 任务中的表名是否包含 schema
- Oracle schema 是否使用大写
- SQL Server 是否使用 `dbo`

### 6.4 脚本执行成功，但平台里没看到测试连接

这是正常现象。`smoke-write-target.ps1` 默认会自动清理临时连接与任务。

如果希望保留资源：

```powershell
.\scripts\smoke-write-target.ps1 ... -KeepResources
```
