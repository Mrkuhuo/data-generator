# 多数据源模拟数据生成器设计规范

## 设计目标

这是一个中文后台工具，不是营销站点。
界面必须优先服务以下核心任务：

1. 配置目标数据库连接
2. 选择或创建目标表
3. 定义字段与模拟数据生成规则
4. 配置写入计划
5. 查看写入结果与校验信息

设计应参考 Airtable 一类“结构化数据工作台”的气质：
- 清晰
- 克制
- 稳定
- 高信息密度但不拥挤
- 适合长时间使用

## 整体风格

- 采用浅色工作台风格，避免大面积深色背景
- 主色使用偏自然的绿色，代表“数据表”“已连接”“可执行”
- 背景使用轻微层次，不使用夸张渐变和炫技光效
- 组件边角适中，避免过圆
- 阴影要轻，优先靠边框和分区建立层次
- 所有文案必须是中文，避免乱码和英文混杂

## 布局原则

- 页面结构遵循“标题区 -> 主操作区 -> 结果区”
- 导航应像工作台标签页，而不是营销卡片入口
- 核心表单必须按真实操作顺序排列
- 任务列表、状态卡、执行结果等信息区应保持稳定位置，避免跳动
- 桌面端优先双栏，移动端自动收敛为单栏

## 组件原则

### 导航
- 使用标签页或工作台导航样式
- 当前页面应有明显但克制的高亮
- 导航文案短而明确

### 表单
- 表单要强调“逐步完成”，不要一次堆满
- 一组字段只解决一个问题
- 高级选项默认收起
- 必填与状态信息要清楚

### 面板
- 面板像数据工作区，而不是展示海报
- 标题和说明分层明显
- 支持在同一个页面里并排对比配置与结果

### 状态
- 成功、警告、失败的反馈颜色要稳定统一
- 调度状态、任务状态、执行状态应有统一的徽标体系
- 状态文案要直接，例如“运行中”“已暂停”“未调度”

### 数据展示
- 执行结果优先展示关键指标
- 写入前条数、写入后条数、净变化、空值校验要直接可见
- JSON 和日志区应保持等宽字体与明显的背景区分

## 文案规范

- 以动词开头，减少抽象表达
- 优先使用“连接”“写入”“预览”“执行”“暂停”“恢复”
- 避免“任务定义中心”“智能配置台”这类空泛命名
- 对用户的提示必须具体，可直接操作

## 禁止项

- 不要做成科技感宣传页
- 不要使用大面积紫色、霓虹、玻璃拟态
- 不要让按钮样式过于花哨
- 不要把同类状态用不同语言表达
- 不要在后台工具里使用过重动画

## 当前项目特别要求

- 写入任务页必须遵循：
  连接与表 -> 字段映射 -> 写入计划 -> 预览与执行结果
- 如果选择已有表，字段应自动映射
- 如果新建表，字段应支持新增和删除
- 持续写入、单次定时、周期定时都应在“写入计划”区统一配置
- 执行结果必须突出写入前后条数、实际写入条数和空值校验
